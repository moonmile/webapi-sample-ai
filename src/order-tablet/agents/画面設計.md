
- 背景色：グレー（`bg-gray-800`）
- テキスト：白

## 画面 1：ホーム（`/`）

### 概要

来店時に最初に表示される画面。来店人数を設定して注文を開始する。

### レイアウト

```
┌─────────────────────────────────────────┐
│         （グラデーション背景: 赤→オレンジ）  │
│                                         │
│    🍣 日経寿司へようこそ                   │
│    心を込めたお寿司をお楽しみください       │
│                                         │
│  ┌────────────────────────────────────┐ │
│  │  テーブル番号: T-001               │ │
│  │                                    │ │
│  │  何名様でお越しですか？              │ │
│  │                                    │ │
│  │     [−]    4    [+]               │ │
│  │                                    │ │
│  │       [ 注文開始 →]                │ │
│  └────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

### UI 要素

| 要素 | 内容 | 備考 |
|------|------|------|
| 背景 | 赤→オレンジのグラデーション | `from-red-600 to-orange-500` |
| テーブル番号 | `T-001`（ハードコード） | 実運用では動的取得 |
| 人数セレクタ | 1〜8名、デクリメント・インクリメントボタン | デフォルト 2 |
| 注文開始ボタン | 白背景・赤テキスト | クリックで `/products` へ遷移 |

### インタラクション

1. `[−]` ボタン押下 → guests 減少（最小 1）
2. `[+]` ボタン押下 → guests 増加（最大 8）
3. 「注文開始」押下 → sessionStorage に `guests` 保存 → `/products` へ遷移

## 画面 2：商品一覧（`/products`）

### 概要

数値ボタン（キーパッド）で商品番号を入力して商品を選択する。
「注文追加」ボタンで数量入力ダイアログを表示し、「OK」でカートに追加する。

```

│  🍣 お寿司注文                       🛒 カート（3）→    │
│  テーブル: T-001                      （赤ヘッダー）     │
├─────────────────────────────────────────────────────────┤
│                                                         │
│              商品番号                                    │
│         ┌──────────────┐                               │
│         │   No.031     │  ← 入力中（一致なし: 赤枠）   │
│         └──────────────┘                               │
│         商品番号が正しくありません                        │
│                                                         │
│         ┌──────┐ ┌──────┐ ┌──────┐                   │
│         │  1   │ │  2   │ │  3   │                   │
│         ├──────┤ ├──────┤ ├──────┤                   │
│         │  4   │ │  5   │ │  6   │                   │
│         ├──────┤ ├──────┤ ├──────┤                   │

│                  └──────┘ └──────┘                   │
│                                                         │
├─────────────────────────────────────────────────────────┤
│              [ 注文追加 ]  （常に表示・赤ボタン）         │

```

### 数量入力ダイアログ

```
┌──────────────────────────┐
│     数量を入力            │
│  ┌────────────────────┐  │
│  │ No.001  まぐろ      │  │
│  │        ¥150        │  │
│  └────────────────────┘  │
│  ┌────────────────────┐  │
│  │    1  個           │  │
│  └────────────────────┘  │
│  [ 1 ] [ 2 ] [ 3 ]       │
│  [ 4 ] [ 5 ] [ 6 ]       │
│  [ 7 ] [ 8 ] [ 9 ]       │
│        [ 0 ] [削除]       │
│  [ キャンセル ] [  OK  ]  │
└──────────────────────────┘
```

### UI 要素

| 要素 | 内容 | 備考 |
|------|------|------|
| ヘッダー | 赤背景、カートバッジ | カート数がリアルタイム反映 |
| 商品番号ディスプレイ | `No.___` 形式（最大3桁） | 一致商品あり: 赤枠・商品名と価格を表示 |
| キーパッド | 1〜9・0・削除 の数値ボタン | 削除は1桁ずつ消去 |
| 注文追加ボタン | 赤ボタン | 画面下部に常時固定表示 |
| 数量ダイアログ | 選択商品情報・数量キーパッド・OK/キャンセル | 数量0は OK 不可 |

### インタラクション

1. ホーム画面から `/products` へ遷移
2. 数値ボタンで商品番号を入力（最大3桁）
   - 一致する商品がある場合 → ディスプレイを赤枠にして商品名・価格を表示
3. 「削除」ボタン → 1桁ずつ削除
4. 「注文追加」押下
   - 商品番号が存在しない場合 → 「商品番号が正しくありません」エラー表示
   - 正常時 → 数量入力ダイアログを表示
5. ダイアログ内でキーパッドを使って数量を入力
6. 「OK」押下
   - 数量が 0 の場合 → 「数量を入力してください」エラー表示
   - 正常時 → カートに追加（localStorage 更新）・ダイアログを閉じる・入力欄リセット
7. 「キャンセル」押下 → ダイアログを閉じる
8. 「🛒 カート」押下 → `/cart` へ遷移
9. API エラー時 → ダミーデータで表示継続（エラーは無視）

## 画面 3：カート（`/cart`）

### 概要

カート内の商品確認と数量調整を行い、注文を確定する。

### レイアウト

```
┌─────────────────────────────────────────────────────────┐
│  [← 戻る]  カートを確認する          （赤ヘッダー）      │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  テーブル: T-001  |  人数: 4名                          │
│                                                         │
│  ┌─────────────────────────────────────────────────┐   │
│  │ まぐろ           ¥580       [−] 2 [+]   [🗑]   │   │
│  │ サーモン         ¥480       [−] 1 [+]   [🗑]   │   │
│  │ えび             ¥380       [−] 3 [+]   [🗑]   │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  合計: 3品   ¥2,300                                    │
│                                                         │
│  ┌───────────────────────────────────────────────┐     │
│  │           注文を確定する →                     │     │
│  └───────────────────────────────────────────────┘     │
│                                                         │
│  （エラー時）❌ 注文に失敗しました: ...                  │
└─────────────────────────────────────────────────────────┘
```

### UI 要素

| 要素 | 内容 | 備考 |
|------|------|------|
| テーブル情報 | テーブル番号・人数 | sessionStorage から取得 |
| カートリスト | 商品名・単価・数量調整・削除 | リアルタイム小計更新 |
| 合計表示 | 品数・合計金額 | — |
| 確定ボタン | グラデーション赤ボタン | 送信中はスピナー表示 |
| エラーメッセージ | 赤背景で表示 | API エラー時のみ |

### インタラクション

1. `[−]` / `[+]` 押下 → 数量変更（localStorage 更新）
2. `[🗑]` 押下 → 商品削除（localStorage 更新）
3. 「注文を確定する」押下
   - カートアイテム数分だけ `POST /api/orders` を順次実行
   - 成功 → orderData を localStorage に保存 → `/order-complete` へ遷移
   - 失敗 → エラーメッセージ表示

## 画面 7：レビュー投稿（`/reviews`）

### 概要

食事後の評価・コメントを投稿する画面。CSRF トークンで保護されたフォーム。

### レイアウト

```
┌─────────────────────────────────────────────────────────┐
│  レビューを投稿する                   （ローズテーマ）    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│  お食事はいかがでしたか？                               │
│  ぜひご感想をお聞かせください                           │
│                                                         │
│  評価                                                   │
│  ☆ ☆ ☆ ☆ ☆   → ★★★☆☆（選択後）                    │
│                                                         │
│  コメント ＊                                            │
│  ┌─────────────────────────────────────────────────┐   │
│  │ お食事のご感想をお聞かせください                  │   │
│  └─────────────────────────────────────────────────┘   │
│                                                         │
│  追加情報（任意）                                       │
│  席番号: [____]  注文番号: [____]  商品番号: [____]     │
│                                                         │
│  ┌───────────────────────────────────────────────┐     │
│  │             レビューを送信する                   │     │
│  └───────────────────────────────────────────────┘     │
│                                                         │
│  （成功時）✅ レビューを送信しました！ありがとうございます│
│  （エラー時）❌ 送信に失敗しました: ...                  │
└─────────────────────────────────────────────────────────┘
```

### UI 要素

| 要素 | 内容 | 備考 |
|------|------|------|
| 星評価 | 1〜5 のボタン | 選択した星以下を色付き表示 |
| コメント | テキストエリア（必須） | 最低 1 文字 |
| 席番号 | 数値入力（任意） | `seat_id` として送信 |
| 注文番号 | 数値入力（任意） | `order_id` として送信 |
| 商品番号 | 数値入力（任意） | `product_id` として送信 |
| 送信ボタン | ローズ色ボタン | CSRF トークン取得後に活性化 |
| ステータス表示 | 成功/エラー | 送信後に表示 |

### CSRF 処理フロー

```
画面表示時
  └─ GET /sanctum/csrf-cookie
        └─ Cookie に XSRF-TOKEN がセット
              └─ フォームが活性化（isReady = true）

送信時
  └─ X-XSRF-TOKEN ヘッダーに Cookie 値をセット
        └─ POST /api/reviews
```

### インタラクション

1. 星ボタン押下 → 評価値設定（1〜5）
2. コメント入力 → フォーム値更新
3. 「レビューを送信する」押下
   - バリデーション（コメント必須）
   - CSRF トークン確認
   - `POST /api/reviews` 送信
   - 成功 → 成功メッセージ表示（フォームリセット）
   - 失敗 → エラーメッセージ表示
