# 前提

- OS: Amazon Linux 2023
- 権限: ec2-user で sudo 実行可能
- ソース配置先: /var/www/laravel-webapi
- SSL: Let’s Encrypt を使う場合、事前にドメインの A レコードを EC2 に向ける

# 0. セキュリティグループ

- インバウンド: 80/tcp, 443/tcp, 22/tcp（必要最小の送信元に制限）
- アウトバウンド: フル許可で可（npm/composer取得などに利用）

# 1. OS 更新と Docker/Compose 導入

```bash
sudo dnf update -y
sudo dnf install -y docker
# compose-plugin が未導入の場合
# sudo dnf install -y docker-compose-plugin || true

sudo systemctl enable --now docker
sudo usermod -aG docker ec2-user
# グループ反映のため一度シェル再ログイン
```

docker-compose-plugin が未導入の場合、手動インストール:
```bash
# バージョンは必要に応じて調整
VERSION=v2.24.7
sudo mkdir -p /usr/local/lib/docker/cli-plugins
sudo curl -SL https://github.com/docker/compose/releases/download/${VERSION}/docker-compose-linux-x86_64 \
  -o /usr/local/lib/docker/cli-plugins/docker-compose
sudo chmod +x /usr/local/lib/docker/cli-plugins/docker-compose

# 動作確認
docker compose version
```

2. プロジェクト配置


```bash
sudo mkdir -p /var/www/laravel-webapi
sudo chown ec2-user:ec2-user /var/www/laravel-webapi
cd /var/www/laravel-webapi

# 例: Git から取得
# git clone https://<your-repo>.git .

# あるいは scp でローカルからアップロード済みを前提
```

3. 環境変数ファイル .env 作成


.env.prod.example を .env にリネームし、必要に応じて編集
```bash
cp .env.prod.example .env
```

.env を用意し、少なくとも以下を設定（RDS 利用なら RDS のエンドポイント/ユーザ/パスワードに置き換え）:

```bash
APP_ENV=production
APP_DEBUG=false
APP_URL=https://your.domain
APP_KEY=   # 後で artisan key:generate で埋める

DB_CONNECTION=mysql
DB_HOST=<rds-endpoint-or-db-service>
DB_PORT=3306
DB_DATABASE=<db_name>
DB_USERNAME=<db_user>
DB_PASSWORD=<db_pass>
```

4. 本番用 compose と Dockerfile を配置
プロジェクトルートに以下ファイルを準備（例）:

- Dockerfile（php-fpmベース）
- docker-compose.prod.yml（app/web/mysql サービス、必要なら mysql を削除して RDS を使用）
- docker/nginx/default.conf（Nginx リバプロ設定）

※ 上記は先に提示したサンプルをそのまま利用可能。RDS を使う場合は compose の mysql サービスを削除し、.env の DB_HOST を RDS に設定。

5. ビルド＆起動

```bash
# ビルド・起動
docker compose -f docker-compose.prod.yml up -d --build

# APP_KEY 生成（未設定の場合）
docker compose exec app php artisan key:generate --force
```

6. マイグレーション（必要ならシードも）

```bash
docker compose exec app php artisan migrate --force --seed
```

7. 動作確認

- ブラウザ/ Postman で http://<EC2 Public IP or Domain>/api/... を確認
- ログ確認

8. SSL (Let’s Encrypt) を EC2 で終端する場合
Nginx を 80/443 で公開した状態で:

```bash
# Certbot インストール（al2023 用 EPEL から）
sudo dnf install -y certbot python3-certbot-nginx
sudo certbot --nginx -d your.domain --redirect
# 自動更新は certbot が systemd timer で登録されることを確認
sudo systemctl list-timers | grep certbot
```

ALB で終端する場合は、ACM 証明書を ALB に設定し、ターゲットグループを EC2:80 に向ける。

9. 自動起動設定（推奨）

方法A: compose の restart ポリシー

- docker-compose.prod.yml の各サービスに restart: unless-stopped を設定（サンプルに含め済み）

方法B: systemd unit

/etc/systemd/system/docker-compose-laravel.service を作成:

```ini
[Unit]
Description=Docker Compose Laravel WebAPI
After=network-online.target docker.service
Wants=network-online.target docker.service

[Service]
Type=oneshot
WorkingDirectory=/var/www/laravel-webapi
ExecStart=/usr/bin/docker compose -f docker-compose.prod.yml up -d
ExecStop=/usr/bin/docker compose -f docker-compose.prod.yml down
RemainAfterExit=yes

[Install]
WantedBy=multi-user.target
```

有効化:

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now docker-compose-laravel.service
```

10. ログローテーション
docker-compose.prod.yml に logging オプション（json-file で max-size: "10m", max-file: "3"）を設定済みか確認。長期保管が必要なら CloudWatch Logs 送信も検討（awslogs ドライバ）。

11. バックアップと監視

- DB: RDS なら自動バックアップを有効化。自己ホスト MySQL なら mysqldump + S3 か EBS スナップショットを定期実行。
- 設定: .env, docker-compose*.yml, Nginx conf を安全な場所にバックアップ。
- 監視: ディスク使用率、コンテナヘルスチェック（compose の healthcheck）、SSL 有効期限、アプリログのエラー件数を定期確認。必要なら CloudWatch アラームを設定。
