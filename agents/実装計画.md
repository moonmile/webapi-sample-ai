## 3. 管理画面でユーザーにより振り分け

管理画面（order-manager）で、ログインユーザーにより以下のように振り分けを行う。

- 管理者ユーザー: admin@example.com
- 店舗ユーザー  : test@example.com

ログイン後の画面遷移を以下のようにする。
店舗ユーザーは、注文の状態画面のみアクセス可能
管理者ユーザーは、商品情報、カテゴリ情報の編集画面のみアクセス可能

## 6. 感想（レビュー）api の実装

レビュー投稿用の API エンドポイントを Laravel 側に実装する。
/api/reviews エンドポイントを作成し、POST メソッドでレビュー情報を受け取る。
CSRF トークンを必要とする。

- openapi.yaml の更新
- ddl.sql の更新
- ReviewController の作成
- Review モデルの作成
- マイグレーションファイルの作成
- ルーティングの設定

## 7. タブレットから感想（レビュー）投稿

タブレット（order-tablet）からレビュー投稿を行う機能を実装する。
