# ログイン認証を実現するためのプランニング

対象のプロジェクト
- src/order-manager/ : React
- src/laravel-webapi/ : Laravel Web API

## 認証方式の選定

- ユーザー名（メールアドレス）とパスワードでログインをする。
- ログインした後は、Bearer トークンを使って認証を行う。
- トークンの発行と管理は Laravel Sanctum を利用する。

## order-manager の実装(React)

1. トップ画面を作成し、メールアドレスとパスワードを入力できるようにする。
2. ログインボタンを押すと、Laravel Web API のログインエンドポイントにリクエストを送信する。
3. ログインに成功した場合、返却された Bearer トークンを保存する
4. ログインに成功すると、厨房管理 /orders に遷移する。

## laravel-webapiの実装(Laravel)

1. ユーザー認証用の API エンドポイントを作成する。
2. Sanctum を利用して Bearer トークンの発行と管理を行う。
3. 注文API エンドポイントで、Bearer トークンを使った認証を実装する。

- Sanctum 保護は orders エンドポイントのみに適用する。

# Further Considerations

- サンプルプログラムのため、トークン有効期限は1時間程度とする。手動で失効させる。
- ロール/権限などはスコープにいれない。
- トークン保持はローカルストレージを利用する。
- クライアントのログアウトで Sanctum トークンを失効する
